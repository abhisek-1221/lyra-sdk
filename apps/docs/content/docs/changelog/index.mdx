<Updates>
  <Update label="v2.0.0">
    ### Transcript API

    * `transcribeVideo(id, options?)` тАФ fetch captions for a single video without an API key
    * `transcribePlaylist(id, options?)` тАФ batch fetch transcripts for every video in a playlist
    * `listCaptionTracks(id)` тАФ discover available language tracks before downloading
    * Output formatters: `toSRT()`, `toVTT()`, `toPlainText()`
    * `TranscriptClient` тАФ reusable client with shared config

    ### Batch Transcript

    * Configurable `concurrency` (1тАУ20 workers) for parallel transcript fetching
    * `from` / `to` range filtering to process a subset of the playlist
    * `onProgress` callback for real-time progress tracking
    * Partial failure handling тАФ individual video failures are caught and recorded, not thrown
    * Returns per-video results with `status: "success" | "failed"`

    ### Caching

    * `InMemoryCache` тАФ process-only cache with TTL and size limits (~0.03ms hits)
    * `FsCache` тАФ filesystem persistence with JSON storage (~0.3ms hits)
    * `CacheStore` interface for custom backends (Redis, etc.)
    * Automatic cache key generation per video ID, language, and metadata flag

    ### Retry & Resilience

    * Exponential backoff retry for transient HTTP failures (429, 5xx)
    * `retries` and `retryDelay` options on all transcript requests
    * `AbortSignal` support to cancel in-flight retry loops

    ### Comments API

    * `comments(videoId, opts?)` тАФ fetch all comment threads with auto-pagination
    * `topComments(videoId, limit?)` тАФ top comments sorted by relevance
    * `commentReplies(commentId)` тАФ fetch all replies to a specific comment
    * `commentsWithReplies(videoId)` тАФ threads + all replies in one call
    * `searchComments(videoId, query)` тАФ keyword search within comments
    * `commentStats(videoId, threads)` тАФ aggregate stats (unique authors, reply ratio, most liked)
    * `commentQuery(videoId)` тАФ fluent builder for filtering and slicing comments

    ### Internationalization (i18n)

    * `regions(hl?)` тАФ list all supported YouTube regions (110+)
    * `languages(hl?)` тАФ list all supported languages (80+)
    * `videoCategoriesByRegion(regionCode, hl?)` тАФ fetch region-specific video categories
    * Localized names via `hl` parameter (e.g., `ja` for Japanese)

    ### REST API

    * New endpoints: `/api/comments/{videoId}`, `/api/comments/{videoId}/top`, `/api/comment-replies/{id}`
    * New endpoint: `/api/transcript/{id}` тАФ single video transcript (no API key)
    * New endpoint: `/api/playlist/{id}/transcript` тАФ batch playlist transcript
    * Removed URL utility endpoints (moved to standalone `lyra-sdk/url` subpath)

    ### Framework Examples

    * Added Hono, NestJS, and Next.js API server examples alongside Express
     * API Reference now shows framework-specific code samples (Express, Hono, Next.js)

  </Update>

  <Update label="v2.0.2">
    ### New Features

    * Added playlist and video retrieval capabilities with concurrent fetching and enriched metadata тАФ `getPlaylist` and `getVideos` now support batched concurrent requests with full snippet enrichment
    * Exposed URL parsing and formatting utilities via new public entry points (`lyra-sdk/url`, `lyra-sdk/fmt`) тАФ no API key required

    ### Bug Fixes

    * Fixed 404 error messages for comment endpoints тАФ `resourceIdForParams` now includes `videoId` and `allThreadsRelatedToChannelId` so `getVideoComments` and `getChannelComments` produce the correct resource ID in error context
    * Added validation for `concurrency` parameter in `hydrateMissingReplies` тАФ non-finite, zero, or negative values are normalized to prevent silent no-ops and `RangeError` crashes
    * Normalized `maxResults`/`maxItems` to safe integers in comment pagination тАФ prevents `NaN` and fractional values from being sent to the YouTube Data API
    * Added validation for cached transcript data to prevent stale entries тАФ cache entries are now validated for structural integrity before serving
    * Added `TranscriptOptions` union fallback overload to `transcribeVideo` тАФ callers passing the base `TranscriptOptions` type no longer fail overload resolution

    ### Improvements

    * Improved retry logic with exponential backoff for transient API failures тАФ configurable `retries` and `retryDelay` on all transcript and comment requests
    * Enhanced type inference for transcript requests with explicit `TranscriptOptions` overload in `transcribeVideo`
    * Implemented concurrency limits for efficient reply fetching in `hydrateMissingReplies` тАФ configurable per-call concurrency parameter
    * Added fail-fast validation for `YOUTUBE_API_KEY` in batch transcript scripts with clear error messages and `.env` setup hints
    * Added explicit Vitest imports to `entrypoints.test.ts` for consistency with other test files
    * Aligned `biome.json` `$schema` version to match minimum installed `@biomejs/biome` version (2.4.12)

    ### Breaking Changes

    * None тАФ fully backwards-compatible with v2.0.0
  </Update>

  <Update label="v1.1.0">
    ### Playlist Query

    * Added `POST /api/playlist/{id}/query` endpoint for filtering, sorting, and slicing playlist videos
    * Filter by duration, views, and likes with min/max ranges
    * Sort by views, likes, or duration in ascending or descending order
    * Range parameter to paginate results (1-indexed start/end)

  </Update>

  <Update label="v1.0.0">
    ### Video API

    * `getVideo(id)` тАФ fetch full video details by ID or URL
    * `getVideos(ids)` тАФ batch fetch multiple videos
    * `getVideoTitle(id)` тАФ lightweight title-only lookup (1 quota unit)

    ### Channel API

    * `getChannel(id)` тАФ fetch channel metadata by ID, @username, or URL
    * `getChannelVideos(id, limit)` тАФ fetch recent uploads for a channel

    ### Playlist API

    * `getPlaylist(id)` тАФ fetch complete playlist with all videos and total duration
    * `getPlaylistInfo(id)` тАФ playlist metadata only (1 quota unit)
    * `getPlaylistIds(id)` тАФ fetch all video IDs with auto-pagination

    ### URL Utilities

    * `parseUrl(url)` тАФ parse a YouTube URL into a structured result
    * `extractUrl(url, type?)` тАФ extract video, playlist, or channel IDs from a URL

    ### Express REST API

    * Full REST API wrapper around all SDK functions
    * Interactive OpenAPI documentation at `/docs/api-reference`
    * Local development server via `npm run dev:express`

  </Update>
